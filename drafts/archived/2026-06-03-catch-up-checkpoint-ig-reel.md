## Concept

Use the four-day panic as B-roll, then land the punch with the catch-up checkpoint logic.

## Structure

1. **Opening B-roll:** terminal logs, checkpoint files, stale queue/state views.
   - On-screen text: `task complete`
   - On-screen text: `task pending`

2. **Problem:** "Hermes had two versions of reality, and neither one crashed."
   - Show logs and memory state side by side.

3. **Wrong instinct:** "I wanted to build the elegant fix."
   - On-screen text: `versioning`
   - On-screen text: `Merkle trees`
   - On-screen text: `audit trails`

4. **Actual fix:** "I built a catch-up checkpoint."
   - Show session history being read.
   - Show reconstructed state as the final source of truth.

5. **Punch:** "The working solution was the clever one."

## Caption

Hermes broke silently.

Logs said one thing.
Memory said another.

The fix was not a smarter resolver.

It was a catch-up checkpoint that rebuilt state from the full session history.

Sometimes the boring source of truth is the architecture.
