Kept reaching for the strongest model on every call, even the boring ones. That habit gets expensive fast, even running local.

The shift that mattered: stop treating the top-tier model as the workhorse. Treat it as the planner.

Let it decide the structure, the "here's what needs to happen." Hand execution, the repetitive low-stakes calls, to something lighter. A 7B model doesn't need to be a genius to run a drift check. It needs to be fast and cheap.

Once you ask "what's the minimum model that gets this job done" on every call instead of every pipeline, the whole architecture gets cheaper without getting dumber.
