# X Thread: Memory Needs an Expiration Date
# Type: Tutorial
# Target Window: 2026-W23

1/ Most people think agent memory is a moat.

To be honest, if your agent remembers everything, it will eventually understand nothing.

2/ The phase before production humbles you is:

"more context = better output"

That works right up until stale context starts making decisions for you.

3/ Then the weird stuff starts.

Old decisions outrank current ones.
Temporary experiments come back like policy.
The agent quotes something from 10 days ago with full confidence.

4/ That is not intelligence.

That is drift.

5/ A memory system is not complete until it knows what to forget.

6/ In Hermes, I treat memory less like a scrapbook and more like a system:

- durable facts
- volatile notes
- context that should decay unless something reaffirms it

7/ Because the real failure mode is not that the agent forgot.

It is that it remembered the wrong thing with high confidence.

8/ If your system cannot tell the difference between a core decision, a stale note, and a dead experiment, you have not built memory.

You have built a hoard.

9/ The next phase of agent engineering is memory governance:

retention
pruning
contradiction handling
provenance
forgetting on purpose

10/ That is the difference between a demo that feels smart and a system that stays useful.

Are you building recall, or are you building judgment?
