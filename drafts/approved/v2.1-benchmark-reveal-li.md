# v2.1 Benchmark Reveal: LinkedIn Post (July 15)

## Copy-Paste Ready

Three weeks ago, voice-match scored 0.6447 on its own benchmark. Ten test cases. Five of them failing hard.

Today it scores 0.871. Zero failures. Same benchmark. Same ten cases. Same rules.

Here's what happened.

I built a benchmark for voice-match: ten social voice cases, each with reject traits (em dashes, marketing hype, corporate polish) and required traits (conspiratorial tone, technical honesty, punchy hooks).

Run 1: 0.6447. Half the cases were failing because the model kept slipping em dashes into output and quoting input verbatim with reject traits still baked in.

The post-processor fixed the em dashes deterministically. Strip and replace. Zero tolerance. That moved the needle to 0.79.

But the real gain came from the skill itself. I rewrote the rules: stronger anchoring on punchy/conspiratorial tone, expanded reject trait list, removed the "Original Hook" section that was causing input quoting.

Score jumped to 0.871. Zero failures across all ten cases.

The delta: +35% from original baseline. Deterministic compliance: zero em dashes, zero reject traits. Same 14B local model. Zero API cost.

This is what "skills evolve" actually looks like. Not vague promises. Numbers. Benchmarks. Public code.

The post-processor is a 47-line Python script. The skill update was a targeted rewrite of three rules. The benchmark runs locally with Ollama. Total cost: $0.0139 in compute.

Phase 4 starts tomorrow. The Harvester goes live on Twitter. Governance gates hold or they don't. We'll find out.
