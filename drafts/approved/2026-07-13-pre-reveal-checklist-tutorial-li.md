The night before a public reveal, I ran one final check. Not on the code. On the claims.

Every number on the landing page, every score in the blog post, every metric in the thread — traced back to a source file. Not "approximately right." Exactly right.

Here's the checklist:
1. Every quantitative claim has a source file path in the commit
2. Every benchmark score matches the latest run, not the first one
3. Every "zero-cost" claim is backed by actual token counts, not assumptions
4. Every governance claim has a test that proves it

Three numbers failed the check. Fixed them tonight. Would have shipped wrong tomorrow.

The lesson: the code can be perfect and the narrative can be lying. Check both.
