An agent looked at my repo and confidently described architecture that didn't exist. Not vague, not hedged. Specific file names, specific functions, stated as fact.

That's the moment Karen got built.

Four rules came out of it: no plan in the same turn as unverified analysis, confidence tags on every claim, a hard stop before any file edit, and explicit reporting when a file just isn't there instead of guessing.

Karen is two local models running adversarial review against each other before anything ships. Not because I don't trust AI agents. Because I've watched one sound completely certain while being completely wrong, and certainty is not the same thing as being right.

If you're building with agents and you don't have something checking their work, you're trusting confidence as a proxy for accuracy. Don't.
